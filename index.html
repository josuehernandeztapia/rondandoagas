1. Disparador de Datos

Nuestra IA (HASE) detecta una anomalía operativa (ej. caída en consumo de GNV).

2. Intervención Proactiva

Nuestro agente (PIA) contacta al operador vía WhatsApp con un mensaje de apoyo.

3. Activación Flexible

El operador activa "Protección Rodando" con un simple 'Sí', sin fricción.

El algoritmo recalcula el plan de pagos para mantener intacta la rentabilidad del crédito.

Ecosistema de Pagos Sin Fricción

Diseñamos un sistema de cobro que se adapta al flujo de efectivo real del transportista, no al revés. Esto reduce drásticamente la fricción y la probabilidad de impago.

1. Recaudo Automático vía Ahorro

Una parte de la cuota se cubre con el ahorro real generado por el consumo de GNV. La unidad se paga sola, en parte, mientras trabaja.

Tecnología Clave: Integración vía API y Webhooks con estaciones de GNV aliadas.

2. Pago Flexible del Remanente

El operador paga el resto con total comodidad: vía transferencia SPEI o en efectivo en +20,000 tiendas de conveniencia.

Tecnología Clave: Integración con pasarelas (STP, OXXO Pay) para conciliación automática.

Riesgos y Mitigación: Nuestro Modelo como Escudo

Hemos diseñado nuestro negocio para convertir los riesgos tradicionales del sector en fortalezas competitivas.

Riesgo: Alta Morosidad (>8%)

Mitigación: Nuestro Scoring Operativo (HASE) y la Cobranza Predictiva (PIA) nos permiten actuar antes de la mora. No reaccionamos, prevenimos.

Mitigación: Convertimos esta dependencia en nuestro foso. Las alianzas y acuerdos exclusivos con estaciones GNV crean una barrera de entrada para competidores.

Modelo Dinámico de Probabilidad de Default ($P_{default}$)

A diferencia de los modelos estáticos, nuestro motor HASE trata la $P_{default}$ como una variable dinámica en el tiempo. La clave es el Flag de Intervención, que reduce matemáticamente el riesgo.

$P_{default}(t) = f(\text{Estáticos}, \text{Score}_{\text{Conductual}}(t-1), \text{Flag}_{\text{Intervención}}(t))$

Anexo B: Roadmap Tecnológico Evolutivo

Nuestro plan de desarrollo es pragmático y de capital eficiente, evolucionando de un MVP ágil a una plataforma propietaria y escalable.

Fase 0: Piloto Lean (Mes 1-6)

Objetivo:

MVP Defendible. Validar hipótesis clave con 15-20 unidades y un stack ultra-lean.

Stack Tecnológico Clave:

  • Automatización: Make.com, Zapier
  • Base de Datos: Google Sheets, Airtable
  • Scoring: API de terceros (benchmark)

Fase 1: Consolidación (Mes 7-18)

Objetivo:

Desarrollar el core propietario de scoring y automatización, escalando a 35 unidades.

Stack Tecnológico Clave:

  • Backend: Python (FastAPI)
  • Streaming: Apache Kafka
  • IA: XGBoost, Scikit-learn
  • Base de Datos: PostgreSQL

Fase 2: Arquitectura de Escala (Mes 19+)

Objetivo:

Expandir a +75 unidades con una infraestructura robusta, BI avanzado y MLOps.

Stack Tecnológico Clave:

  • Orquestación: Kubernetes (EKS/GKE)
  • Streaming Avanzado: Apache Flink
  • MLOps: MLflow, Kubeflow
  • Seguridad: HashiCorp Vault
